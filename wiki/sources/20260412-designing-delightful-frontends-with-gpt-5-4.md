# Summary

This guide details techniques for steering GPT-5.4 toward polished, production-ready frontend designs. It covers model improvements, guidance strategies, prompt patterns, and the use of design principles to generate distinctive user interfaces. The document includes a dedicated `frontend-skill` to help users get visually ambitious results and avoid common design pitfalls.

# Key Claims

- **GPT-5.4 is notably better for frontend web development**: It generates more visually appealing and ambitious designs compared to previous versions, emphasizing UI and imagery.
- **Model improvements enable more complex, complete apps**: Improved image understanding, native tool use (image search/generation, Playwright), and verification capabilities result in more functional, reliable interfaces.
- **Guidance is essential**: Without specific prompting, GPT-5.4 may fall into generic patterns. Clear, constraining instructions yield higher quality, distinctive outputs.
- **Practical prompt techniques**: Setting reasoning levels, defining design systems, providing visual references, and structuring page content as narratives all materially improve output.
- **The 'frontend-skill' consolidates these best practices**: This skill provides hard design rules, establishes good defaults, and supports both landing pages and app UIs.
