# Definition
Skill packages are curated sets of guidance, prompts, and hard/soft rules that can be installed or referenced by LLMs to execute specific task domains (e.g., frontend design, code review).

# Key Ideas
- Enforce deliberate design structure and taste
- Provide defaults for composition, imagery, copy, and motion
- Prevent common errors/mediocrity in model generations

# Variants
- Standalone skills (like Frontend Skill)
- Combined multi-skill packs

# Open Questions
- How can skill packages remain up-to-date with evolving design standards?
